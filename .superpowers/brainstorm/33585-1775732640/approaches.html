App Architecture — 3 Approaches

How should the modular scale tool be structured?

A

Single-Page App (Recommended)

Next.js App Router + single route

Everything lives on one page — controls on the left/top, preview + export on the right/bottom. URL params sync all state. No routing needed.

Pros

  • Fastest to build and iterate
  • Instant feedback — tweak a value, see it immediately
  • URL sharing is trivial (one page = one URL)
  • Simple deployment on Vercel
  • Perfect for a focused tool like this

Cons

  • All complexity on one page (manageable for this scope)
  • Mobile layout needs careful thought
B

Multi-Page with Tabs

Separate pages: Calculator → Preview → Export

Each concern gets its own page/tab. Calculator page for inputs, Preview page for type specimen, Export page for code output.

Pros

  • Each page is simpler individually
  • More room for elaborate previews
  • Cleaner mobile experience per page

Cons

  • Breaks the instant feedback loop — changing a value means switching back to see it
  • More routing complexity for URL state
  • Unnecessary for this scope
C

Dashboard-Style with Panels

Resizable panels: sidebar controls + main preview + bottom export

IDE-like layout with draggable panel dividers. Maximum information density.

Pros

  • Power-user friendly
  • All information visible simultaneously
  • Feels professional and sophisticated

Cons

  • Significantly more complex to build (resizable panels, responsive breakpoints)
  • Intimidating for newcomers
  • Overkill for a focused calculator tool

Shared Architecture (All Approaches)

Tech Stack

  • Next.js 15 App Router
  • TypeScript
  • shadcn/ui + Tailwind CSS
  • nuqs for URL state management

Core Engine

  • lib/scale.ts — pure math functions
  • Double-strand merge algorithm
  • Rounding to 1px / 2px grid
  • Extractable for Figma plugin later

Features

  • 1 or 2 base values (merged sort)
  • Named ratio presets + free input
  • Configurable -N to +N steps
  • Google Fonts picker for preview

Output

  • CSS custom properties
  • SCSS variables + map
  • Tailwind config
  • JSON export
  • Configurable variable prefix
